**Human Explanation Pack**
Issue: `BitArray` could store zero-bit shot data, but public conversion methods crashed on it.

Expected behavior: an empty classical register still has shots, so counts should preserve shot multiplicity as `{"": shots}` or `{0: shots}`.

Root cause: the code flattened shots with `reshape(-1, 0)`, which NumPy cannot infer. String conversion also used `bin(0).zfill(0)`, which returns `"0"` instead of `""`.

Fix: flatten using the known `self.size * self.num_shots` count. For explicit `num_bits=0` samples, build an empty `(shots, 0)` byte array after validating values fit into zero bytes.

Tests: `test_empty_register_counts` proves counts and bitstrings preserve shots for zero-bit registers. Constructor tests prove `from_counts` and `from_samples` can produce zero-bit `BitArray` values.

Likely reviewer question: why not return `{}`?  
Answer: maintainer discussion says the counts values should still sum to total shots, even for an empty register.

Known limitation: this focuses on the reported public count/bitstring/sample paths, not a full audit of every `BitArray` method with `num_bits=0`.
