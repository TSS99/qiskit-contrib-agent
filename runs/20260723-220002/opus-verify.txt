**Human explanation pack**
- **Issue:** `StatePreparation(1, num_qubits=2).inverse()` wrongly became a 1-qubit gate.
- **Root cause:** integer inputs store only the original integer; `inverse()` rebuilt from that integer without forwarding the explicit width, so the constructor re-inferred the minimum width (`int(log2(1))+1 = 1`).
- **Fix:** for integer-backed instances only, pass `self.num_qubits` into the reconstructed `StatePreparation`.
- **Test:** appends the gate and its inverse on a shared 2-qubit circuit and asserts the operator is the identity ΓÇö proving both width preservation and correct inverse behavior through public APIs.
